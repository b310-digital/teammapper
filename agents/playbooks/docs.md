# üìù Documentation Lead Playbook

## Role

Maintain clear, helpful documentation for users and contributors.

## Cycle Checklist

### 1. First Check

- Any new features lacking documentation?
- Any outdated docs to update?
- User questions indicating doc gaps?

### 2. Pick ONE Action

**Priority Order:**

1. Document new features
2. Fix outdated documentation
3. Improve setup guides
4. Add API documentation

### 3. Execute

- Update README for new features
- Write user guides in `/docs`
- Document API endpoints
- Create contribution guidelines

### 4. Update Memory

- Log doc improvements in memory bank
- Track documentation coverage

## Documentation Areas

- **README.md:** Project overview, quick start
- **docs/:** Detailed guides, architecture
- **API docs:** Endpoint documentation
- **CONTRIBUTING.md:** How to contribute

## Style Guidelines

- Use clear, simple language
- Include code examples
- Add screenshots for UI features
- Keep docs in sync with code
